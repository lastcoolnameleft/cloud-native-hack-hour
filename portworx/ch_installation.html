Environment preparation¶

Deploy Portworx to your Kubernetes cluster using the Operator approach. Different environments have different requirements for enabling dynamic drive management in Portworx:

  • Azure requires a service principal and secret to be created.

Deploy PX-Enterprise to Cluster-1¶

To install you will generate a spec file and modify it before applying it.

Generate Spec File¶

** Open your browser and navigate to https://central.portworx.com

  1. Login with the account you created previously

  2. Select “Portworx Enterprise” and click next in the top section of the Spec Generator

  3. On the next page, select “Use the Portworx Operator”, Portworx Version = 2.7, and ETCD “Built-In”. Then click “Next”

  4. The following page allows you to configure your cloud environment: Select the following:

  1. Select your environment: “Cloud”

  2. Select Cloud Platform: “AZURE”

  3. Configure storage devices: Select “Create Using a Spec”, then under “Select Volume Type” choose “Standard” and leave the size at 150GB. Click Next.

  1. On the Networking Screen, Click Next to accept the defaults

  2. On the Customize page: “Are use running on either of these?” : Choose “Azure Kubernetes Service (AKS)”. Then expand the “Advanced Settings” and check “Enable Monitoring”. Click Next.

  3. On the final page, you can provide a name and tags to save the spec file for later access and use. Please do so.

Note

On this page, there are links to install Portworx, but it will place the installation in the kube-system namespace. On managed solutions like AKS, we recommend installing to a different namespace. Follow these instructions to retrieve your spec file and change the namespace.

Install the operator:

kubectl apply -f 'https://install.portworx.com/2.7?comp=pxoperator'

Download your Saved Spec from Portworx Central:

curl -Lo px-spec.yaml https://install.portworx.com/2.7?operator=true&mc=false&kbver=&b=true&s=%22type%3DStandard_LRS%2Csize%3D150%22&kd=type%3DPremium_LRS%2Csize%3D150&c=px-cluster-8fe65c3e-621d-4fc6-b951-c480c62061fc&aks=true&stork=true&csi=true&mon=true&st=k8s&promop=true

Note

For mac users, please install gnu-sed to make your life easier: gnu-sed installation instructions

Note

The following instructions are to be executed on either Linux or MacOS via CLI. The values can also be changed with a text editor. If you choose to edit in a text editor, verify your work before you apply the spec file.

Change the namespace:

sed -i 's/kube-system/portworx/g' px-spec.yaml

Change the prometheus endpoint:

sed -i 's/\/\/prometheus/\/\/px-prometheus/g' px-spec.yaml

Set the webhook for stork to true:

sed -i 's/webhook-controller: \"false\"/webhook-controller: \"true\"/g' px-spec.yaml

Deployment¶

And finally, apply the px-spec.yaml spec file:

kubectl apply -f px-spec.yaml

Watch as the cluster initializes¶

Once you have applied the spec file you can watch cluster initialization events:

watch "kubectl get events --field-selector involvedObject.kind=StorageCluster -n portworx --sort-by .lastTimestamp"

When the cluster is done initializing you will see a ClusterOnline event.

Install the Portworx kubectl CLI - PXC¶

Finally, install the pxc cli utility along with the pxc-pxctl component.

Note

** Run the following commands from the home directory of the folder where you extracted the zip file **

For Linux:

curl -Lo /tmp/pxc.tar.gz https://github.com/portworx/pxc/releases/download/v0.33.0/pxc-v0.33.0.linux.amd64.tar.gz
gunzip /tmp/pxc.tar.gz
tar -xf /tmp/pxc.tar -C /tmp
mv /tmp/pxc/kubectl-pxc /usr/local/bin/kubectl-pxc
chmod +x /usr/local/bin/kubectl-pxc
cp pxc-pxctl /usr/local/bin/pxc-pxctl
chmod +x /usr/local/bin/pxc-pxctl
alias pxctl='kubectl pxc pxctl'

For OSX:

curl -Lo /tmp/pxc.tar.gz https://github.com/portworx/pxc/releases/download/v0.33.0/pxc-v0.33.0.darwin.amd64.tar.gz
gunzip /tmp/pxc.tar.gz
tar -xf /tmp/pxc.tar -C /tmp
mv /tmp/pxc/kubectl-pxc /usr/local/bin/kubectl-pxc
chmod +x /usr/local/bin/kubectl-pxc
cp pxc-pxctl /usr/local/bin/pxc-pxctl
chmod +x /usr/local/bin/pxc-pxctl
alias pxctl='kubectl pxc pxctl'

Initialize the utility to use the portworx namespace¶

kubectl-pxc config cluster set --portworx-service-namespace=portworx

Check that the pxc cli is working correctly¶

Use the pxctl alias to get the cluster status:

pxctl status

Use the pxc cluster describe command:

kubectl pxc cluster describe
