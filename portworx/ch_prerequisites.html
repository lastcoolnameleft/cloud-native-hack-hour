Portworx Microsoft Azure Instructions¶

This document will walk you through today’s hands-on experience with Portworx, by Pure Storage.

The sections below outline a series of steps that map to a test plan you can track using the Portworx_POC_Plan.xlsx spreadsheet in this repository.

All tests in the plan have a corresponding section of this document with the instructions on how to run the test and how to validate the results.

Kubernetes artifacts for the different tests are available in the px-poc/yamls folder and are based on public repository container images. For air-gapped clusters, there is a script provided to upload these images to your registry and to modify the yaml artifacts to use your repository url and credentials.

The test plan is focused on kubernetes and is validated against all major kubernetes distributions.

Prerequisites¶

Two Kubernetes clusters with network connectivity between them. Each cluster should have the following resources at a minimum:

  • 4 VMs (Single master and 3 workers nodes)

  • 4 VCPUs and 16 GB of RAM on worker nodes

  • 50 GB for base OS

  • 2 x 100GB drives for each of the worker nodes.

For more details on networking/operating system requirements please refer to the Portworx documentation.

Access to an object store endpoint to test backup/restore functionality.

  • 100GB of available space

If an objectstore is not available, instructions on deploying Minio are included in the installation section.
