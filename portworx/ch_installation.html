Environment preparation¶

Deploy Portworx to each Kubernetes cluster using the Operator approach. Different environments have different requirements for enabling dynamic drive management in Portworx:

  • Azure requires a service principal and secret to be created.

Deploy PX-Enterprise to Cluster-1¶

To install you will generate a spec file and modify it before applying it.

Generate Spec File¶

Let’s start with a basic spec file for your environment. To do that we’re going to set some environment variables:

Note

Edit these variables to make sure to pick the correct disk type and cloud type setting for your environment!

# st the version of Portworx to install
PXVER=2.7.0
# choose from gp2|io1|ssd|standard|premium|zeroedthick (valid storage type for your cloud provider)
disk_type=zeroedthick
# change this to higher size if you are doing benchmarking
disk_size=150
# choose from eks|aks|gke|pks or leave empty
env=

Now that you have chosen the correct settings based on your environment you can generate and download the spec file:

curl -Lo px-spec.yaml "https://install.portworx.com/$PXVER?operator=true&mc=false&kbver=&b=true&s=%22type%3D${disk_type}%2Csize%3D${disk_size}%22&kd=type%3Dgp2%2Csize%3D150&c=px-cluster&eks=true&stork=true&mon=true&st=k8s&promop=true"

curl -Lo px-spec.yaml "https://install.portworx.com/$PXVER?operator=true&mc=false&kbver=&b=true&kd=%2Fdev%2F${etcd_drive}&c=px-cluster&stork=true&mon=true&st=k8s&promop=true"

Modify Spec File

Note

For mac users, please install gnu-sed to make your life easier: gnu-sed installation instructions

Change the namespace:

sed -i 's/kube-system/portworx/g' px-spec.yaml

Change the prometheus endpoint:

sed -i 's/\/\/prometheus/\/\/px-prometheus/g' px-spec.yaml

Set the webhook for stork to true:

sed -i 's/webhook-controller: \"false\"/webhook-controller: \"true\"/g' px-spec.yaml

Deployment¶

Once you have created a spec file you can deploy the operator:

kubectl create ns portworx
curl -Lo px-operator.yaml https://install.portworx.com/?comp=pxoperator
sed -i 's/kube-system/portworx/g' px-operator.yaml
kubectl create -f px-operator.yaml

And finally, apply the px-spec.yaml spec file:

kubectl apply -f px-spec.yaml

Air-gapped Deployment¶

Set your registry location:

# set your registry location (my-registry.my-company.com for example)
REGISTRY_LOCATION=my-registry.my-company.com

And append the registry location setting to your spec file:

echo "  customImageRegistry: $REGISTRY_LOCATION" >> px-spec.yaml

If the registry requires credentials we need to create a registry secret.

Edit these variables with your correct information:

DOCKER_REGISTRY_SERVER=my.registry.com
DOCKER_USER=username
DOCKER_PASSWORD=password
DOCKER_EMAIL=username@company.com

Use the following command to create the docker registry secret:

kubectl create secret docker-registry regcred -n portworx \
    --docker-server=$DOCKER_REGISTRY_SERVER \
    --docker-username=$DOCKER_USER \
    --docker-password=$DOCKER_PASSWORD \
    --docker-email=$DOCKER_EMAIL

And append the registry secret setting to our spec file:

echo "  imagePullSecret: regcred" >> px-spec.yaml

Download the operator spec and modify it to deploy to the portworx namespace:

kubectl create ns portworx
curl -Lo px-operator.yaml https://install.portworx.com/?comp=pxoperator
sed -i 's/kube-system/portworx/g' px-operator.yaml
kubectl apply -f px-operator.yaml

Modify the specification file to use your registry and deploy it:

kubectl patch serviceaccount portworx-operator -n portworx -p '{"imagePullSecrets": [{"name": "regcred"}]}'
sed -i "s/image: portworx/image: ${REGISTRY_LOCATION}\/portworx/g" px-operator.yaml
kubectl apply -f px-operator.yaml

And finally, apply the px-spec.yaml spec file:

kubectl apply -f px-spec.yaml

Watch as the cluster initializes¶

Once you have applied the spec file you can watch cluster initialization events:

watch "kubectl get events --field-selector involvedObject.kind=StorageCluster -n portworx --sort-by .lastTimestamp"

When the cluster is done initializing you will see a ClusterOnline event.

Install the Portworx kubectl CLI - PXC¶

Finally, install the pxc cli utility along with the pxc-pxctl component.

Note

Run the following commands from the home directory of the git repository.

For Linux:

curl -Lo /tmp/pxc.tar.gz https://github.com/portworx/pxc/releases/download/v0.33.0/pxc-v0.33.0.linux.amd64.tar.gz
gunzip /tmp/pxc.tar.gz
tar -xf /tmp/pxc.tar -C /tmp
mv /tmp/pxc/kubectl-pxc /usr/local/bin/kubectl-pxc
chmod +x /usr/local/bin/kubectl-pxc
cp pxc-pxctl /usr/local/bin/pxc-pxctl
chmod +x /usr/local/bin/pxc-pxctl
alias pxctl='kubectl pxc pxctl'

For OSX:

curl -Lo /tmp/pxc.tar.gz https://github.com/portworx/pxc/releases/download/v0.33.0/pxc-v0.33.0.darwin.amd64.tar.gz
gunzip /tmp/pxc.tar.gz
tar -xf /tmp/pxc.tar -C /tmp
mv /tmp/pxc/kubectl-pxc /usr/local/bin/kubectl-pxc
chmod +x /usr/local/bin/kubectl-pxc
cp pxc-pxctl /usr/local/bin/pxc-pxctl
chmod +x /usr/local/bin/pxc-pxctl
alias pxctl='kubectl pxc pxctl'

Initialize the utility to use the portworx namespace¶

kubectl-pxc config cluster set --portworx-service-namespace=portworx

Check that the pxc cli is working correctly¶

Use the pxctl alias to get the cluster status:

pxctl status

Use the pxc cluster describe command:

kubectl pxc cluster describe

Install the Stork CTL¶

On Linux:

STORK_POD=$(kubectl get pods -n portworx -l name=stork -o jsonpath='{.items[0].metadata.name}') &&
kubectl cp -n portworx $STORK_POD:/storkctl/linux/storkctl ./storkctl
sudo mv storkctl /usr/local/bin &&
sudo chmod +x /usr/local/bin/storkctl

On OSX:

STORK_POD=$(kubectl get pods -n portworx -l name=stork -o jsonpath='{.items[0].metadata.name}') &&
kubectl cp -n portworx $STORK_POD:/storkctl/darwin/storkctl ./storkctl
sudo mv storkctl /usr/local/bin &&
sudo chmod +x /usr/local/bin/storkctl
